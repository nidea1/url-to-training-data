## ğŸ¨ Design Patterns Used

1. **Strategy Pattern**
   - Different cleaning strategies per domain
   - Different chunking strategies per domain

2. **Dependency Injection**
   - Pipeline receives configured components
   - Components receive configuration

3. **Factory Pattern**
   - Config classes create instances
   - Pipeline creates component instances

4. **Facade Pattern**
   - Pipeline provides simple interface
   - Hides complex component interactions

5. **Template Method**
   - Data generation follows template
   - Scraping follows template
